Converting a Qucs Schematic to a VeroRoute Netlist


Valid Components

A selection of useful Qucs components was chosen (see list below) and a schematic containing them all drawn called qucs-netlist.sch. The components were then associated with gEDA VeroRoute components as there is a reasonable selection of them and their component names are valid in VeroRoute, so a netlist using them can be imported by VeroRoute. Component sizes were not considered as Qucs components haven’t got sizes – it’s not really relevant on a schematic. The size of imported components can be adjusted in VeroRoute anyway. The Qucs libraries could have special components added for VeroRoute, for different sizes for example.


The program could be used to create a netlist for a different PCB layout application if that accepted VeroRoute component names, or a datafile with acceptable component names was created.


The Qucs schematic file is used to find the components


The Qucs parts library is here /usr/local/share/qucs/library/*.lib


There is a little information here https://qucs-help.readthedocs.io/en/latest/internal.html?highlight=create%20net


File contents

Irrelevant rows are skipped until the section that starts with the line <Components>. This section contains the circuit components of the schematic. The conversion ends with the line </Components>.

Example line <R R1 1 50 20 -26 15 0 0 "50 Ohm" 1 "26.85" 0 "0.0" 0 "0.0" 0 "26.85" 0 "european" 0>

The Qucs sim file is used to find the nets

The Python script produces this from the schematic file by using the Qucs Snap with the command

qucs-spice.qucs -i inputfile.sch -o outputfile.sim -n


File contents

The file does not contain hidden components. Lines between .Def:name to .Def:End are irrelevant and skipped. The rest contain netlist details.

Example line R:R8 _net0 _net1 R="50 Ohm" Temp="26.85" Tc1="0.0" Tc2="0.0" Tnom="26.85"


The line format is:

The first field contains Qucs component type and name separated by a colon. Then netnames, they are followed by component details in the format xxx=xxxx.
